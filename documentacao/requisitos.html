Requisitos do Sistema de Controle de Serviços Técnicos

Requisitos Funcionais

Os requisitos funcionais descrevem as funcionalidades que o sistema deve oferecer aos usuários.

RF1: Gestão de Usuários

RF1.1: Cadastrar usuários com nome, email e senha
RF1.2: Definir nível de acesso (admin ou usuário)
RF1.3: Autenticar usuários no sistema
RF1.4: Gerenciar permissões baseadas no nível de acesso
RF1.5: Registrar data de cadastro automaticamente

RF2: Gestão de Cargos

RF2.1: Cadastrar cargos com nome e descrição
RF2.2: Consultar cargos disponíveis
RF2.3: Atualizar informações de cargos

RF3: Gestão de Colaboradores

RF3.1: Cadastrar colaboradores com dados pessoais
RF3.2: Associar colaborador a um cargo
RF3.3: Vincular colaborador a um usuário do sistema
RF3.4: Consultar colaboradores cadastrados
RF3.5: Atualizar informações de colaboradores

RF4: Gestão de Clientes

RF4.1: Cadastrar clientes (pessoa física ou jurídica)
RF4.2: Registrar dados de contato (telefone, email)
RF4.3: Armazenar endereço do cliente
RF4.4: Consultar clientes cadastrados
RF4.5: Atualizar informações de clientes

RF5: Gestão de Ordens de Serviço

RF5.1: Criar nova ordem de serviço
RF5.2: Gerar número de OS automaticamente
RF5.3: Registrar equipamento e defeito relatado
RF5.4: Vincular OS a um cliente
RF5.5: Atribuir técnico responsável
RF5.6: Registrar serviço realizado
RF5.7: Calcular valor total do serviço
RF5.8: Atualizar status da OS (aberta, em andamento, concluída, cancelada)
RF5.9: Consultar ordens de serviço

RF6: Relatórios

RF6.1: Gerar relatório de ordens de serviço
RF6.2: Gerar relatório de clientes
RF6.3: Gerar relatório de colaboradores
RF6.4: Gerar relatório de cargos
RF6.5: Gerar relatório de usuários

RF7: Impressão

RF7.1: Imprimir ordem de serviço
RF7.2: Imprimir relatórios gerados
RF7.3: Visualizar prévia de impressão

Requisitos Não Funcionais

Os requisitos não funcionais descrevem as características e restrições do sistema que não estão diretamente relacionadas às funcionalidades.

RNF1: Segurança

RNF1.1: Armazenar senhas criptografadas
RNF1.2: Controle de acesso baseado em níveis de usuário
RNF1.3: Validação de dados nos formulários
RNF1.4: Proteção contra injeção SQL
RNF1.5: Sanitização de dados de entrada

RNF2: Usabilidade

RNF2.1: Interface intuitiva e responsiva
RNF2.2: Feedback visual para ações do usuário
RNF2.3: Mensagens de erro claras e informativas
RNF2.4: Atalhos para funções frequentes
RNF2.5: Layout consistente em todas as telas

RNF3: Desempenho

RNF3.1: Tempo de resposta rápido para consultas
RNF3.2: Otimização de consultas ao banco de dados
RNF3.3: Carregamento eficiente de páginas

RNF4: Confiabilidade

RNF4.1: Tratamento adequado de exceções
RNF4.2: Backup automático do banco de dados
RNF4.3: Recuperação de falhas

RNF5: Manutenibilidade

RNF5.1: Código organizado e documentado
RNF5.2: Arquitetura modular (classes separadas)
RNF5.3: Padrões de codificação consistentes

RNF6: Compatibilidade

RNF6.1: Compatibilidade com navegadores modernos
